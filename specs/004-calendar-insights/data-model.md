## New Entities

### InsightEntity *(new — DB v3)*

A single persisted pattern or correlation observation generated by the analysis engine.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | `Long` | PK, auto-generate | |
| `dataType1Id` | `Long` | FK → `data_types.id` CASCADE DELETE | First data type in the correlation pair |
| `dataType2Id` | `Long` | FK → `data_types.id` CASCADE DELETE | Second data type in the correlation pair |
| `optionId` | `Long?` | Nullable, FK → `multiple_choice_options.id` CASCADE DELETE | Set when the insight involves a specific MC option |
| `correlationCoefficient` | `Double` | Not null | -1.0 to +1.0 |
| `correlationMethod` | `String` | Not null | `"PHI"`, `"POINT_BISERIAL"`, `"PEARSON"`, `"MEAN_COMPARISON"` |
| `insightText` | `String` | Not null | Human-readable insight sentence with emoji |
| `sampleSize` | `Int` | Not null | Number of days analysed |
| `computedAt` | `Long` | Not null | Epoch millis when this insight was generated |

**Cascade behaviour**: Deleting either referenced data type (or a referenced MC option) automatically removes all insights involving it. This satisfies the edge case: "data type deleted after insights generated".

**Filtering invariant**: Only insights where `|correlationCoefficient| ≥ 0.35` (mapping to ~60% co-occurrence per FR-015) are persisted. The analysis engine filters before insert.

```kotlin
@Entity(
    tableName = "insights",
    foreignKeys = [
        ForeignKey(
            entity = DataTypeEntity::class,
            parentColumns = ["id"],
            childColumns = ["dataType1Id"],
            onDelete = ForeignKey.CASCADE
        ),
        ForeignKey(
            entity = DataTypeEntity::class,
            parentColumns = ["id"],
            childColumns = ["dataType2Id"],
            onDelete = ForeignKey.CASCADE
        )
    ],
    indices = [
        Index(value = ["dataType1Id"]),
        Index(value = ["dataType2Id"]),
        Index(value = ["computedAt"])
    ]
)
data class InsightEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val dataType1Id: Long,
    val dataType2Id: Long,
    val optionId: Long? = null,
    val correlationCoefficient: Double,
    val correlationMethod: String,
    val insightText: String,
    val sampleSize: Int,
    val computedAt: Long = System.currentTimeMillis()
)
```

### AnalysisMetadataEntity *(new — DB v3)*

Single-row table tracking when the last analysis was performed and how much data existed at that time. Used to determine whether re-analysis is needed (FR-011).

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | `Int` | PK, always `0` | Single-row pattern |
| `lastAnalysisTimestamp` | `Long` | Not null, default 0 | Epoch millis of last completed analysis |
| `lastDataCount` | `Int` | Not null, default 0 | Sum of `daily_entries` + `multi_choice_selections` row counts at time of analysis |

**Re-analysis logic**: On Calendar screen open, query current total entry count. If `currentCount != lastDataCount` → re-analyse. If equal → load persisted insights from `insights` table.

```kotlin
@Entity(tableName = "analysis_metadata")
data class AnalysisMetadataEntity(
    @PrimaryKey val id: Int = 0,
    val lastAnalysisTimestamp: Long = 0,
    val lastDataCount: Int = 0
)
```

### InsightRepository

Wraps `InsightDao` and `AnalysisMetadataDao`. Owns the "should we re-analyse?" decision logic and the insert/clear lifecycle.

```kotlin
class InsightRepository(
    private val insightDao: InsightDao,
    private val analysisMetadataDao: AnalysisMetadataDao,
    private val dailyEntryDao: DailyEntryDao,
    private val multiChoiceSelectionDao: MultiChoiceSelectionDao
) {
    fun getTopInsightsFlow(limit: Int = 10): Flow<List<InsightEntity>> =
        insightDao.getTopInsightsFlow(limit)

    suspend fun replaceAllInsights(insights: List<InsightEntity>) {
        insightDao.deleteAll()
        insightDao.insertAll(insights)
    }

    suspend fun needsReanalysis(): Boolean {
        val meta = analysisMetadataDao.getMetadata()
        val currentCount = getCurrentDataCount()
        return meta == null || meta.lastDataCount != currentCount
    }

    suspend fun recordAnalysisCompleted() {
        val count = getCurrentDataCount()
        analysisMetadataDao.upsert(
            AnalysisMetadataEntity(
                id = 0,
                lastAnalysisTimestamp = System.currentTimeMillis(),
                lastDataCount = count
            )
        )
    }
```

Pure Kotlin class (no Android dependencies) that takes raw entry data and computes pairwise correlations. Stateless — receives data, returns results.

```kotlin
    fun analyseAll(
```
